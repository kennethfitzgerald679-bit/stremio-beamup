# beamup-sync

Clean rewrite of the Beamup backup/sync tooling.

## Files

- `beamup-sync.sh`: main CLI
- `beamup-backup.sh`: local backup creator
- `beamup-restore.sh`: local restore command
- `beamup-common.sh`: shared helpers

## Commands

```bash
beamup-sync config
beamup-sync verify
beamup-sync backup
beamup-sync push [--force] [--all] [--ftp|--s3|--rsync]
beamup-sync pull <archive-name|--latest> [--restore] [--ftp|--s3|--rsync]
beamup-sync list [--ftp|--s3|--rsync]
beamup-sync restore [archive-path] [--force]
```

Options:

- `-v, --verbose`: verbose output
- `-f, --force`: `push` creates backup first, `restore` skips prompt
- `-a, --all`: push all local archives
- `-r, --restore`: pull then restore

## Config file

Path: `/etc/beamup/sync.conf` (generated by `beamup-sync config`)

Format: shell variables (quoted strings), for example:

```bash
BEAMUP_BASE="/var/backups/beamup"
LOCAL_ARCHIVE_DIR="/var/backups/beamup/archives"
DOWNLOAD_DIR="/var/backups/beamup/downloads"
LOG_DIR="/var/log/beamup-sync"
LOCK_FILE="/var/lock/beamup-sync.lock"
RETENTION_DAYS="30"
BACKUP_ENCRYPTION_ENABLED="false"
BACKUP_ENCRYPT_SSH_KEY="/etc/beamup/keys/sync"
ENABLED_REMOTES="s3,rsync"

FTP_ENABLED="false"
FTP_HOST=""
FTP_PORT="21"
FTP_USER=""
FTP_PASSWORD=""
FTP_REMOTE_PATH="/backups"
FTP_VERIFY_TLS="true"

S3_ENABLED="true"
S3_BUCKET="my-bucket"
S3_REGION="us-east-1"
S3_ACCESS_KEY="..."
S3_SECRET_KEY="..."
S3_PREFIX="beamup"
S3_ENDPOINT_URL=""
S3_VERIFY_SSL="true"
S3_AUTO_CREATE_BUCKET="true"

RSYNC_ENABLED="true"
RSYNC_MODE="ssh"
RSYNC_HOST="backup.example.com"
RSYNC_PORT="22"
RSYNC_USER="backup"
RSYNC_REMOTE_PATH="/backups"
RSYNC_SSH_KEY="/root/.ssh/id_rsa"
RSYNC_PASSWORD=""
```

## Behavior notes

- Commands require root.
- Remote operations are non-interactive.
- Rsync supports both transports:
  - `RSYNC_MODE="ssh"`: `user@host:path` over SSH
  - `RSYNC_MODE="daemon"`: `rsync://user@host:873/module/path`
- `RSYNC_PORT` is used by both modes (typically `22` for SSH, `873` for daemon).
- For SSH mode, auth order is SSH key first, then password fallback (`sshpass`) if configured.
- For daemon mode, `RSYNC_PASSWORD` is passed via rsync's standard `RSYNC_PASSWORD` env var.
- S3 push auto-creates the bucket when missing if `S3_AUTO_CREATE_BUCKET="true"`.
- If `BACKUP_ENCRYPTION_ENABLED="true"`, backups are encrypted with `age` using the SSH private key in
  `BACKUP_ENCRYPT_SSH_KEY` and saved as `beamup-backup-<timestamp>.tar.xz.age`.
- If encryption is disabled, backups are saved as `beamup-backup-<timestamp>.tar.xz`.
- Every archive (encrypted or plain) has a matching `.sha256` checksum file.
- Backup content is mandatory and always includes:
  - `/home/dokku/.ssh/authorized_keys`
  - `/home/dokku` (Dokku app directories)
  - `/etc/ssh/ssh_host_*`
  - `/etc/cron.daily` (and `/etc/crontab` if present)
- Backups exclude Dokku app cache directories (`/home/dokku/*/cache/*`).
- Restore flow creates missing Dokku apps first, restores `/home/dokku` without `.ssh`,
  restores `/etc/ssh/ssh_host_*` and restarts SSH, then applies ownership/perms.
- Restoring an encrypted archive (`*.tar.xz.age`) automatically decrypts it using
  `BACKUP_ENCRYPT_SSH_KEY`.
